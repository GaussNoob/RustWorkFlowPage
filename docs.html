Documentação RustWorkflow

Guia completo de uso do editor visual de automação com IA, motor nativo Rust e 33 tipos de nós. v1.0.0

O que é o RustWorkflow?

RustWorkflow é um editor visual de automação baseado em nós (node-based). Você cria fluxos arrastando blocos no canvas, conecta-os com linhas, e executa tudo com um clique.

IA Integrada

Chat com LLMs locais (Ollama) ou cloud (OpenAI, Anthropic) diretamente no fluxo.

Motor Rust

Engine nativo em Rust para tarefas de alta performance (HTTP, arquivos).

REST & SQL

Faça chamadas HTTP, conecte a MySQL e PostgreSQL, tudo visualmente.

Código Custom

Execute JavaScript ou Rust customizado quando os nós padrão não bastam.

Instalação

Linux

Baixe o arquivo .AppImage na página de download. Depois, torne-o executável e abra:

  1. Baixe o arquivo RustWorkflow-1.0.0.AppImage
  2. Clique com botão direito → Propriedades → Permissões → marque "Executável"
  3. Dê duplo-clique para abrir, ou execute via terminal: ./RustWorkflow-1.0.0.AppImage

Windows

O instalador para Windows estará disponível em breve. Fique atento às atualizações.

Requisitos

  • Ollama (opcional) — Para usar IA local, instale o Ollama e baixe um modelo (ex: ollama pull llama3)
  • Rust (opcional) — Apenas se quiser executar nós de código Rust customizado

Primeiros Passos

Crie seu primeiro fluxo em menos de um minuto:

  1. Abra o RustWorkflow e veja o canvas vazio
  2. Na Sidebar (esquerda), encontre o nó Start e arraste para o canvas
  3. Arraste um nó Debug Log para o lado direito do Start
  4. Conecte-os: clique no ponto de saída do Start e arraste até a entrada do Debug Log
  5. Clique no Debug Log para abrir o Painel de Propriedades (direita) e escreva uma mensagem
  6. Clique em ▶ Executar na barra superior
  7. Veja o resultado no Painel de Logs (inferior)
Pronto! Esse é o fluxo básico: arraste nós, conecte, configure e execute.

Canvas & Editor

O canvas é a área central onde você monta seus fluxos. Recursos:

  • Arrastar e soltar — Arraste nós da Sidebar para o canvas
  • Zoom — Use a roda do mouse ou pinch para aproximar/afastar
  • Pan — Clique e arraste o fundo para mover a visualização
  • Conectar nós — Clique no handle de saída (direita) e arraste até o handle de entrada (esquerda) de outro nó
  • Selecionar — Clique em um nó para ver suas propriedades
  • Deletar — Selecione um nó ou conexão e pressione Delete
  • Minimap — Toggle do minimapa para visão geral do fluxo

Catálogo de Nós 33 tipos

Nós organizados em 5 categorias:

Triggers (3) Actions (10) Logic (4) Data (9) Advanced (7)

Triggers — Iniciam o fluxo

Nó O que faz
Start Ponto de início obrigatório. Todo fluxo começa aqui.
Webhook Recebe requisições HTTP externas para iniciar o fluxo.
Cron Executa automaticamente em intervalos (a cada minuto, hora, dia ou customizado).

Actions — Executam ações

Nó O que faz
HTTP Request Faz chamadas a APIs (GET, POST, PUT, DELETE, PATCH). Headers e body configuráveis.
AI Chat Envia prompts para LLMs — Ollama (local), OpenAI ou Anthropic.
File Read Lê o conteúdo de um arquivo do sistema.
File Write Escreve ou adiciona conteúdo em um arquivo.
Debug Log Imprime uma mensagem no painel de logs.
MySQL Executa queries SQL em bancos MySQL.
PostgreSQL Executa queries SQL em bancos PostgreSQL.
Notification Envia notificações (toast, alert, console, som).
Delay Pausa a execução por tempo definido (ms, segundos, minutos).
Wait Espera por uma condição ou timeout.

Logic — Controle de fluxo

Nó O que faz
If Caminho condicional: saída True ou False baseado numa expressão.
Switch Multi-caminho: avalia um valor e segue o case correspondente (+ default).
Loop Repetição: forEach, while, for, repeat. Saída "body" durante e "done" ao terminar.
Try/Catch Tratamento de erros: saída "try" para sucesso e "catch" para erro.

Data — Transformação de dados

Nó O que faz
Set Variable Define variáveis que outros nós podem usar via {{nome}}.
Math 14 operações: soma, subtração, multiplicação, divisão, raiz, arredondamento, etc.
JSON Parse (texto → objeto) e stringify (objeto → texto).
Array map, filter, reduce, sort, push, find, length e mais.
String uppercase, lowercase, trim, replace, split, join, slice, includes.
DateTime now, format, parse, add/subtract, diff, compare.
Regex match, replace, test, split com expressões regulares.
Template Renderiza templates com variáveis dinâmicas.
Type Convert Converte entre tipos: toString, toNumber, toBoolean, toJSON, toArray.

Advanced — Recursos avançados

Nó O que faz

Variáveis

Variáveis permitem passar dados entre nós. Use a sintaxe {{nome}} em qualquer campo de texto.

Como usar

  • O nó Set Variable cria variáveis: defina nome e valor
  • Qualquer nó pode referenciar variáveis: {{minha_variavel}}
  • Acesse saídas de nós anteriores: {{$node["HTTP Request"].json.data}}
  • Variáveis especiais: {{$now}} retorna o timestamp atual

Autocompletar

Ao digitar {{ em qualquer campo, o editor mostra sugestões automáticas baseadas nos nós anteriores do fluxo. Navegue com as setas ↑↓ e confirme com Enter.

Dica: No painel de Variáveis, arraste o nome de qualquer variável diretamente para um campo de texto para inseri-la.

Condições & Loops

Nó If

Avalia uma expressão e segue um de dois caminhos:

  • Campo 1: valor ou variável da esquerda (ex: {{status}})
  • Operador: equals, not equals, contains, greater than, less than, etc.
  • Campo 2: valor de comparação (ex: 200)
  • Saída True → segue um caminho
  • Saída False → segue outro caminho

Nó Switch

Funciona como um "if" com múltiplas opções. Defina uma expressão e crie N cases com valores. O fluxo segue o case que corresponder (ou o default).

Nó Loop

Repete o caminho "body" N vezes. Quando termina, segue a saída "done".

  • forEach: itera sobre um array
  • for: de X até Y com incremento
  • while: enquanto condição for verdadeira
  • repeat: repete N vezes

IA & Ollama

O nó AI Chat permite integrar LLMs diretamente nos seus fluxos.

Usando Ollama (IA Local)

  1. Instale o Ollama e baixe um modelo: ollama pull llama3
  2. Inicie o Ollama: ollama serve
  3. No RustWorkflow, arraste o nó AI Chat para o canvas
  4. No painel de propriedades, selecione Provider: Ollama
  5. O indicador ficará ● Conectado e os modelos serão listados automaticamente
  6. Escolha o modelo, escreva o prompt e configure a temperatura

Configurações

Parâmetro Descrição
Provider Ollama (local), OpenAI ou Anthropic
Modelo Selecionado automaticamente do Ollama, ou manual para Cloud
System Prompt Instrução de comportamento para o modelo
User Message Mensagem enviada (aceita variáveis {{var}})
Temperatura 0.0 (preciso) a 2.0 (criativo) — slider visual
Max Tokens Limite de tokens na resposta
Top P Controle de amostragem (0.0 a 1.0)
Repeat Penalty Penalidade por repetição (0.0 a 2.0)
Performance: Ollama roda 100% localmente — sem enviar dados para nenhum servidor. Os requests incluem timeout de 60 segundos e retry automático.

HTTP & APIs

O nó HTTP Request faz chamadas a qualquer API REST.

Campo Descrição
URL Endereço da API (aceita variáveis)
Método GET, POST, PUT, DELETE, PATCH
Headers Cabeçalhos HTTP (lista de chave/valor)
Body Corpo da requisição em JSON (para POST/PUT)
Variável de saída Nome da variável que receberá a resposta

A resposta fica disponível como variável para nós seguintes. Use {{$node["HTTP Request"].json}} para acessar os dados retornados.

Banco de Dados

Nós MySQL e PostgreSQL executam queries SQL diretamente no fluxo.

Campo Descrição

Database Nome do banco
Usuário / Senha Credenciais de acesso
Query Comando SQL (aceita variáveis {{var}})
Operação Query, Insert, Update ou Delete

Código Custom

O nó Code permite executar lógica personalizada quando os nós padrão não são suficientes.

JavaScript

Executa diretamente no navegador. Acesse variáveis do workflow e retorne dados:

  • Acesso às variáveis do contexto de execução
  • Retorne um objeto JSON que será a saída do nó
  • Ideal para transformações de dados complexas

Rust

Compila e executa código Rust nativo via Electron. Ideal para:

  • Processamento de alta performance
  • Operações que precisam de segurança de memória
  • Integração com bibliotecas Rust
Nota: Para executar código Rust, é necessário ter o Rust e Cargo instalados na máquina.

Idiomas

O RustWorkflow suporta dois idiomas com troca instantânea:

Clique no botão 🌐 na barra de ferramentas para alternar. A preferência é salva automaticamente e mantida entre sessões.

Salvar & Carregar Workflows

Seus fluxos podem ser salvos e recarregados a qualquer momento:

  • Salvar: clique em 💾 e dê um nome ao workflow
  • Carregar: clique em 📂 e escolha um workflow salvo
  • Listar: veja todos os workflows salvos no seletor
  • Excluir: remova workflows que não precisa mais

Os workflows são armazenados localmente no seu computador.

Perguntas Frequentes

O Ollama precisa estar rodando?

Sim. O Ollama deve estar ativo (ollama serve) para que o nó AI Chat se conecte. Um indicador de status mostra se a conexão está ativa.

Posso usar sem internet?

Sim, para a maioria das funcionalidades. IA local (Ollama), manipulação de arquivos, lógica e variáveis funcionam 100% offline. Apenas nós de HTTP e cloud AI precisam de conexão.

Os dados são enviados para algum servidor?

Não. O RustWorkflow roda 100% na sua máquina. Dados de IA local ficam no seu computador. APIs cloud só são utilizadas se você explicitamente configurar OpenAI ou Anthropic.

Quantos nós posso ter em um fluxo?

Não há limite definido. O editor suporta centenas de nós com zoom, pan e minimap para navegação.

Posso executar fluxos automaticamente?

Sim! Use o nó Cron para agendar execuções automáticas a cada minuto, hora, dia, ou em intervalo customizado.
